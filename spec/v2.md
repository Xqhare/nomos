# Historical Musings about v2 (keep for posterity)

## Expand `key="value with spaces"` Syntax

All tags should support this really.

- `@"location with spaces"` even though I dont really forsee a usecase
- `+"kind with spaces"`
- `#"generic with spaces"`

## Allow Single Line Dependencies

`dep=@'src/main.rs'@4`

Would make a task depend on a task in `src/main.rs` at line 4.

PROBLEM: file editing will move the line number.

SOLUTION: The number defines the number of tasks, in the above example the fourth task found in the file.

This still will break when adding new tasks above the one referenced, but could be a good middle ground.

## Relax allowed File types further

Allow any text file (utf8). As long as the syntax is followed on some line of the file.

## Indentation

Allow both spaces and tabs. 1 tab = 4 spaces.
